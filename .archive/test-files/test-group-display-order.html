🔧 Group Display Order Fix

❌ BEFORE: Groups ordered by creation time (newest first)

Problem: When you create Group 1, Group 2, Group 3, they appear as:

  • Group 3 (created last, appears first)
  • Group 2 (created second, appears second)
  • Group 1 (created first, appears last)

This is confusing because the logical order is reversed!

✅ AFTER: Groups ordered numerically by name

Solution: Groups now appear in logical formation order:

  • Group 1 (first group formed)
  • Group 2 (second group formed)
  • Group 3 (third group formed)

Much more intuitive for users!

🔧 Technical Details

SQL Query Change:

-- BEFORE (ordered by creation time)
ORDER BY g.created_at DESC

-- AFTER (ordered numerically by group name)
ORDER BY 
  CASE 
    WHEN g.name REGEXP '^Group [0-9]+$' THEN 
      CAST(SUBSTRING(g.name, 7) AS UNSIGNED)
    ELSE 999999 
  END ASC,
  g.name ASC

Benefits:

  • ✅ Groups appear in logical formation order (1, 2, 3...)
  • ✅ Handles numeric sorting correctly (Group 10 comes after Group 9, not Group 1)
  • ✅ Falls back to alphabetical sorting for non-standard group names
  • ✅ More intuitive user experience
