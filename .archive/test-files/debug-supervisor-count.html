🔍 Debug Supervisor Count Issue

🐛 Problem:

Dashboard shows 0 supervisors instead of the 7 you uploaded.

🔍 Possible Causes:

  • Upload failed: Supervisors weren't saved to database
  • View issue: API uses v_supervisor_workload view which requires users table
  • Authentication: API call is failing due to auth issues
  • Data mismatch: Supervisors in supervisors table but not in users table

📋 Diagnostic Steps:

  1. Test API Call: Check if /api/users/supervisors returns data
  2. Check Upload: Verify supervisor upload process works
  3. Console Errors: Look for JavaScript errors in browser console
  4. Database Check: Verify data is actually in the database

🔧 Likely Fix:

The API endpoint uses v_supervisor_workload view which requires supervisors to be linked to users table.

If supervisors were uploaded but not properly created as users, they won't appear in the count.
